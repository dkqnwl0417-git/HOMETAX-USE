# 홈택스 수집 프로그램 수정 요약

## 1. 수집 필터링 로직 수정 (`server/hometaxCrawler.ts`, `server/crawler.ts`)
- **[전자신고] 우선 수집**: 제목에 `[전자신고]`가 포함된 경우 무조건 수집 대상으로 포함하도록 수정했습니다.
- **세목 필터링 강화**: `[전자신고]`가 포함된 데이터에 대해 법인세, 부가가치세, 종합소득세, 양도소득세, 상속/증여세 등 세부 세목을 더 정확하게 분류하도록 로직을 보완했습니다.
- **기타 세목 처리**: `[전자신고]` 단어가 없더라도 `파일설명서` 또는 `전산매체 제출요령` 키워드가 포함된 경우 '기타' 세목으로 수집되도록 수정했습니다.

## 2. 상세 페이지 링크 연결 오류 수정 (`server/cloudinaryUpload.ts`)
- **Referrer 정책 우회**: 홈택스 보안 정책으로 인해 메인 화면으로 리다이렉트되는 문제를 해결하기 위해, Referrer를 제거하고 메타 태그 및 `location.replace`를 혼합한 안전한 리다이렉트 방식을 적용했습니다.

## 3. Playwright 실행 환경 문제 해결 (`package.json`, `render.yaml`)
- **의존성 추가**: `package.json`에 `playwright` 패키지를 명시적으로 추가했습니다.
- **빌드 스크립트 수정**: Render 배포 시 Playwright 실행에 필요한 브라우저(Chromium) 및 시스템 라이브러리가 자동으로 설치되도록 `render.yaml`의 빌드 명령어를 수정했습니다.
- **메모리 최적화**: 무료 버전의 메모리 제한을 고려하여 크롤링 시 불필요한 리소스 로드를 최소화하도록 코드를 최적화했습니다.

## 4. 기타 수정 사항
- **데이터 정규화**: 수집된 데이터의 날짜 형식을 일관되게 유지하고, 중복 수집 방지 로직을 강화했습니다.
- **알림 생성**: 신규 데이터 수집 시 알림이 정상적으로 생성되도록 보완했습니다.
