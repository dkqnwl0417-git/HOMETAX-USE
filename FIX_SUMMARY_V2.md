# 홈택스 수집 프로그램 및 매뉴얼 자료실 수정 요약 (v2)

## 1. 내부 매뉴얼 일괄 업로드 기능 추가 (`client/src/pages/ManualFiles.tsx`)
- **다중 파일 선택**: 이제 한 번에 여러 개의 파일을 선택하여 업로드할 수 있습니다.
- **업로드 상태 표시**: 각 파일별로 업로드 진행 상황(대기, 성공, 실패)을 실시간으로 확인할 수 있는 UI를 추가했습니다.
- **진행률 바**: 전체 업로드 진행률을 시각적으로 보여주는 프로그레스 바를 적용했습니다.
- **안정성 강화**: 업로드 중에는 입력창을 비활성화하여 데이터 충돌을 방지했습니다.

## 2. 배포 시 데이터 유지 설정 (`server/db.ts`, `render.yaml`)
- **영구 저장소(Persistent Disk) 적용**: Render 배포 환경에서 데이터를 영구적으로 보관할 수 있도록 `/var/data` 경로에 1GB 크기의 디스크 설정을 추가했습니다.
- **DB 경로 최적화**: 배포 시마다 데이터가 초기화되는 문제를 해결하기 위해, 운영 환경에서는 영구 저장소 경로(`file:/var/data/hometax.db`)를 사용하도록 수정했습니다.
- **자동 테이블 생성**: 서버 시작 시 필요한 테이블이 없으면 자동으로 생성하도록 로직을 보완하여 첫 배포 시에도 오류 없이 작동합니다.

## 3. 기존 기능 유지 및 UI 안정화
- **UI 일관성**: 기존 Tailwind CSS와 Shadcn UI 스타일을 유지하여 화면이 깨지지 않도록 설계했습니다.
- **에러 핸들링**: 업로드 실패 시 사용자에게 정확한 피드백을 제공하도록 에러 메시지 처리를 강화했습니다.
- **이전 수정 사항 포함**: 이전에 완료한 홈택스 [전자신고] 필터링 로직 및 상세 페이지 링크 연결 수정 사항도 모두 포함되어 있습니다.

## 4. 배포 시 주의사항
- **데이터 보존**: 이제 새롭게 배포하더라도 `/var/data`에 저장된 데이터베이스 파일은 삭제되지 않고 유지됩니다.
- **환경 변수**: `DATABASE_URL` 환경 변수를 별도로 설정하지 않아도 기본적으로 영구 디스크를 사용하도록 설정되어 있습니다.
