# 홈택스 크롤링 기능 재구성 계획

## 현재 상황 분석 (2026-04-21)

### 문제점
1. **홈택스 자료실 상세페이지 URL 자동 추출 불가**
   - 원인: JavaScript 기반 동적 로딩 구조
   - 서버 측 크롤링으로는 실제 데이터 접근 불가
   - WebSquare API도 보안 검증(CSRF, 세션)으로 인해 응답 불가

2. **현재 크롤러의 문제**
   - 샘플 데이터(2024년)만 사용 중
   - 실제 최신 게시글 정보 수집 불가
   - 저장되는 URL이 유효하지 않음

### 해결 방안 (사용자 요청)
기존의 "자동 URL 연결" 방식에서 "신규 게시글 감지 및 알림" 방식으로 변경

## 새로운 구현 방식

### Phase 1: 신규 게시글 감지 (자동화)
**목표**: 홈택스에 새로운 게시글이 올라왔는지 자동 감지

**구현 방식** (우선순위 순):
1. **RSS 피드 모니터링** (권장 - 90% 가능)
   - 홈택스 공지사항 RSS 피드 구독
   - 30분~1시간 주기로 폴링
   - 장점: 안정적, 리소스 효율적
   - 단점: 피드 제공 여부 확인 필요

2. **공개 API 폴링** (대체 - 70% 가능)
   - 홈택스 공개 API 활용
   - 주기적 폴링으로 신규 게시글 감지
   - 장점: 상세 정보 획득 가능
   - 단점: API 변경 위험

3. **브라우저 자동화** (최후 - 100% 가능)
   - Puppeteer/Playwright 사용
   - JavaScript 렌더링으로 정확한 데이터 추출
   - 장점: 완벽한 데이터 수집
   - 단점: 서버 리소스 과다 소비, 비용 증가

**감지 조건**:
- 제목에 "[전자신고]" 포함 또는
- 제목에 "전산매체 제출요령" 포함
- 문서 유형: "파일설명서" 또는 "전산매체 제출요령"

**감지 시 처리**:
- 제목, 등록일을 DB에 저장
- 사이트 내 알림(🔔) 자동 생성
- 사용자에게 신규 게시글 알림 표시

### Phase 2: 관리자 URL 수동 입력 (새 기능)
**목표**: 감지된 게시글에 대해 관리자가 정확한 URL 입력

**필요한 기능**:
1. 감지된 게시글 목록 페이지
   - 제목, 등록일, 감지 상태 표시
   - URL 미입력 게시글 강조 표시

2. URL 입력 UI
   - 각 게시글별 URL 입력 필드
   - 유효성 검증 (https:// 시작 확인)
   - 저장 버튼

3. URL 저장 및 관리
   - 입력된 URL을 DB에 저장
   - 사용자가 조회할 때 정확한 URL 제공
   - 수정/삭제 기능

### Phase 3: 사용자 조회 기능 (기존 유지)
- 필터링: 기간, 세금 유형, 문서 유형
- 정렬: 최신순
- 페이지네이션: 20개/페이지
- 제목 클릭: 관리자가 입력한 URL로 이동
